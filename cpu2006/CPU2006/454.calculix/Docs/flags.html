454.calculix Flags

These are flags that appear in the benchmark sources, categorized as to whether they are portability candidates, mandatory, or forbidden.

Sections

Selecting one of the following will take you directly to that section:

Portability Flags

  • -DSPEC_CPU_APPEND_UNDERSCORE
  • -DSPEC_CPU_NO_APPEND_UNDERSCORE
  • -DSPEC_CPU_NAGF95
  • -DSPEC_CPU_NOZMODIFIER
    • -DSPEC_CPU_APPEND_UNDERSCORE
    • (?:^|(?<=\s))[-/]DSPEC_CPU_APPEND_UNDERSCORE(?:=\S*)?(?=\s|$)

    • SPEC_CPU_APPEND_UNDERSCORE indicates that when calling Fortran routines from C, an underscore needs to be appended to the routine name. This is the default on all but Windows.

    • -DSPEC_CPU_NO_APPEND_UNDERSCORE
    • (?:^|(?<=\s))[-/]DSPEC_CPU_NO_APPEND_UNDERSCORE(?:=\S*)?(?=\s|$)

    • Does what you think it does. Or what you think it does after you read the description of DSPEC_CPU_APPEND_UNDERSCORE. It's the default on Windows.

    • -DSPEC_CPU_NAGF95
    • (?:^|(?<=\s))[-/]DSPEC_CPU_NAGF95(?:=\S*)?(?=\s|$)

    • SPEC_CPU_NAGF95 causes character string lengths to be appended in calls, as NAG requires.

    • -DSPEC_CPU_NOZMODIFIER
    • (?:^|(?<=\s))[-/]DSPEC_CPU_NOZMODIFIER(?:=\S*)?(?=\s|$)

    • SPEC_CPU_NOZMODIFIER can be used if your compiler does not implement the C99 standard printf length modifier "z". You'll know you need this flag if validation fails with the file SPECtestformatmodifier_z.txt

Mandatory Flags

  • -ISPOOLES
  • -DSPEC_CPU_WINDOWS
    • -ISPOOLES
    • -ISPOOLES\b

    • This flag is used to ensure that include files in the SPOOLES directory may be used. It is included automatically by object.pm.

    • -DSPEC_CPU_WINDOWS
    • (?:^|(?<=\s))[-/]DSPEC_CPU_WINDOWS(?:=\S*)?(?=\s|$)

    • SPEC_CPU_WINDOWS helps us decide whether to, by default, append an underscore.

    • -DARCH
    • (?:^|(?<=\s))[-/]DARCH(?:=\S*)?(?=\s|$)

    • ARCH was the program's original method of selecting options for appending underscores to routine names. When running in the context of SPEC_CPU, instead use SPEC_CPU_APPEND_UNDERSCORE or SPEC_CPU_NO_APPEND_UNDERSCORE

    • -DBE_CAUTIOUS
    • (?:^|(?<=\s))[-/]DBE_CAUTIOUS(?:=\S*)?(?=\s|$)

    • BE_CAUTIOUS may be useful in debugging, but is not used in benchmarking. If you want to use it, notice that you'll have to edit the sources to remove the hard-coded defines that normally set it to zero.
